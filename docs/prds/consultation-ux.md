# PRD: 診療 UX 改善フェーズ(フェーズ2)

2026-07-21 の実機受入(owt-9)フィードバックに基づく UX・翻訳精度改善。機能面(one_way×2 翻訳・表示階層・コピー)は pass、UX と医療語彙精度が受入保留の理由。発話者=人間(医師・患者)、判断者=医師、という前提は CONTEXT.md を踏襲する。

## 要件(EARS)

### ビデオレイアウト

- **REQ-OWT-15**: 2者通話中、システムは常に相手の映像をメイン全面に、自分の映像を右下の小窓(PiP)に重ねて表示すること。小窓のタップ/クリックでメインと PiP を入れ替えられること。モバイルのセーフエリアを侵さないこと。相手未入室時は既存の待機表示を維持すること。
- **REQ-OWT-16**: 相手がマイクをミュートしている場合、システムはその状態を明示すること。発話中の参加者の映像枠をハイライトすること。
- **REQ-OWT-17**: LiveKit 接続が再接続中の場合、システムは通話画面に再接続中バナーを表示すること。
- **REQ-OWT-18**: モバイル端末では、システムはカメラの前面/背面切替コントロールを提供すること(患部提示ユースケース)。

### セッション設定の通話中変更

- **REQ-OWT-19**: 医師が通話中に患者言語を変更した場合、システムは新しい言語を LiveKit participant attributes 経由で患者端末へ伝搬し、双方の Soniox セッションを新設定で再開すること。attributes の受信値は必ずガード関数で検証すること(未知言語コードは無視)。変更時に発話中の字幕が一度確定・中断されることは許容する。
- **REQ-OWT-20**: システムは医療語彙リスト(`lib/medical-terms.ts` を単一情報源とする)を Soniox context として常に両参加者のセッションに渡すこと。医師がロビーまたは通話中に追加したカスタム語彙は attributes 経由で両端末に伝搬し、次回セッション開始(再起動含む)から適用すること。語彙はメモリ内のみで永続化しないこと。context の仕様・上限は Soniox 公式 docs と突合すること(推測記述をしない)。

### 字幕・コピー

- **REQ-OWT-21**: 全文コピーボタンは医師ロールにのみ表示すること(字幕履歴の閲覧は両ロール可)。
- **REQ-OWT-22**: 字幕表示は次の階層を持つこと: 翻訳メインを大きく・原文をミュート色の小さい下段に / ロール色(医師=teal・患者=slate)の識別 / interim は低透明度 / 字幕サイズ S/M/L 切替 / オーバーレイは直近2発話まで。

## スコープ外(見送り理由の記録)

- 画面共有・テキストチャット(字幕が担う)・録画/録音(永続化禁止の不変則)・バーチャル背景(デモ価値薄)・待機室(認証なしデモの範囲外 — docs/design/production-entry-flow.md に本実装設計あり)

## 受入(owt-9 と統合)

- REQ-OWT-15〜22 の実機確認(PC+iPhone、Tailscale HTTPS)
- 医療語彙 fixture: 「ブスコパン」を含む発話が正しく認識・翻訳表示されること(context 拡充の効果確認)
- フェーズ1の受入項目(翻訳メイン表示・コピー形式)の回帰がないこと
