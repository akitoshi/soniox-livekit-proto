# PRD: 翻訳設計変更(one_way ×2)+会話ログコピー

日付: 2026-07-13 / ステータス: grill確定
関連: CONTEXT.md、docs/specs/one-way-translation-plan.md

## 背景と目的

国際クリニック(自費診療インバウンド患者)向けオンライン診療の参照実装として、医師=日本語話者と患者=外国語話者の1対1診療で「互いの発話が相手の言語の字幕で読める」体験を成立させる。現行の two_way 翻訳は設定2言語以外(特に英語)が話されると翻訳されないため、話者ごとの one_way ×2 に変更する。患者言語は特定言語をハードコードせず、Soniox 対応60言語から選べる構造にする。加えて、診療後のカルテ転記を見据えた会話ログのコピー(メモリ内簡易版)を実装する。

クライアント直結アーキテクチャ(各ブラウザ→Soniox WebSocket、一時 API キー、サーバーエージェントなし)は維持する。

## 要件(EARS 形式)

### 通常時(Ubiquitous)
- **REQ-OWT-01**: システムは常に翻訳を話者ごとの one_way で構成すること(医師セッション `target_language`=患者言語、患者セッション `target_language`=`ja`)。two_way を使用しない。
- **REQ-OWT-02**: 患者言語の選択肢は `lib/languages.ts`(公式 Supported languages 準拠・自国語表記)から生成すること。特定言語をハードコードした分岐を持たない。
- **REQ-OWT-03**: システムは常に `language_hints` として `["ja", "en", <患者言語>]`(重複除去)を渡すこと。
- **REQ-OWT-04**: Soniox 恒久 API キーをクライアントへ露出させないこと(一時キー方式を維持)。モデルは `stt-rt-v5` を明示指定すること。
- **REQ-OWT-05**: 音声・字幕・会話ログをアプリケーション側で永続保存しないこと(メモリ内のみ。localStorage も不可)。

### イベント駆動(Event-driven)
- **REQ-OWT-06**: 患者が患者言語(例: ドイツ語)で話した場合、医師画面に日本語訳字幕が概ね1〜2秒以内に表示されること。
- **REQ-OWT-07**: 話者が文中で英語に切り替えた場合(mid-sentence code-switching 含む)、英語部分も欠落なく受信者の言語へ翻訳されること。
- **REQ-OWT-08**: 医師が日本語で話した場合、患者画面に患者の選択言語の訳が表示されること。患者言語を Soniox 対応の任意言語(例: ベトナム語)に変えても、コード変更なしに機能すること。
- **REQ-OWT-09**: 全文コピー操作をした場合、確定済み会話ログ全文が話者・言語ラベル付きプレーンテキスト(`[医師 ja] …` / `[患者 de→ja] 訳文 (原文: …)` 形式)でクリップボードに入ること。

### 状態駆動(State-driven)
- **REQ-OWT-10**: 非確定(interim)字幕の間はグレー系表示とし、確定で通常表示へ切り替えること(既存挙動踏襲)。
- **REQ-OWT-11**: 相手の発話字幕は翻訳文をメイン・原文を小さく併記で表示すること。自分の発話は原文をメインで表示すること。
- **REQ-OWT-12**: 翻訳トークンにはタイムスタンプが無い前提で UI を実装すること(原文トークンとの1対1対応を仮定しない)。

### 望ましくない挙動(Unwanted behaviour)
- **REQ-OWT-13**: 翻訳トークンが生成されない場合(患者言語=`ja` 選択時など)、原文のみで表示・コピーが破綻しないこと。
- **REQ-OWT-14**: data channel で受信した不正な字幕ペイロードはスキーマ検証で棄却し、画面へ表示しないこと。
- **REQ-OWT-15**: 一時キー発行失敗・Soniox エラー時は、ビデオ通話を止めず字幕のみエラー表示すること(既存挙動維持)。

## 受入条件

| REQ | 受入条件(テスト可能な形) |
|---|---|
| REQ-OWT-01 | `use-soniox-captions` の設定が role に応じた one_way になっている(コードレビュー+2ブラウザ手動確認) |
| REQ-OWT-02 | 言語選択 UI の選択肢数=定数の言語数。定数の全行テーブルテスト green。`grep` で言語コードの分岐ハードコードなし |
| REQ-OWT-03 | 患者言語=de のとき hints が `["ja","en","de"]`、=en のとき `["ja","en"]`(ユニットテスト) |
| REQ-OWT-04 | クライアントコードに `SONIOX_API_KEY` 参照なし。`stt-rt-v5` 明示。 |
| REQ-OWT-05 | 保存系 API 呼び出し・storage 書き込みが存在しない。reload で会話ログが消える(手動) |
| REQ-OWT-06〜08 | 2ブラウザ手動テスト: ja⇄患者言語の双方向+英語混在発話で確認 |
| REQ-OWT-09 | 整形ユーティリティのユニットテスト+手動でペースト確認 |
| REQ-OWT-10〜13 | 2ブラウザ手動テスト+翻訳なしケース(患者言語=ja)の表示確認 |
| REQ-OWT-14 | `isCaptionPayload` 拡張のユニットテスト(不正 role・欠落フィールド棄却) |
| REQ-OWT-15 | SONIOX_API_KEY 未設定で起動→通話は成立し字幕のみエラー表示(手動) |
