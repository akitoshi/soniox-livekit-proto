# ADR 0001: 字幕処理にサーバーエージェントを使わない

- ステータス: 採用
- 日付: 2026-07-13

## コンテキスト

対象は、医師と患者の1対1診療で、各参加者の音声を文字起こし・翻訳して字幕表示する用途である。各ブラウザが自分のマイク音声を一時 API key で Soniox WebSocket へ直接送り、確定字幕を LiveKit で相手へ配信する構成が成立している。音声録音は要件ではない。

## 決定

LiveKit Agents を採用せず、クライアント直結アーキテクチャを維持する。1対1診療・字幕用途では、サーバー側音声処理を挟まない方が遅延、追加コスト、運用複雑性のすべてで優位である。

## 結果

- 音声処理 worker の配備、監視、スケーリングを追加しない。
- Soniox 恒久 API key は API Route だけに置き、ブラウザには短期・単一使用の一時 key だけを渡す。
- 各クライアントが自分の Soniox 接続を持つため、1診療を2ストリームとしてキャパシティ計画する。
- クライアントのバックグラウンド移行やネットワーク断への再接続は、クライアント側の責務として残る。

## 再検討トリガー

次のいずれかが要件化した場合に、この決定を再検討する。

1. 音声録音の保存。
2. 3言語以上の参加者対応（通訳・家族を含む）。
3. 通話中のリアルタイム要約。

再検討は自動的な LiveKit Agents 採用を意味せず、その時点の遅延、コスト、運用、データ保護要件を比較して新しい ADR を作る。
