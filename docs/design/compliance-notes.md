# コンプライアンス注意事項

## 文字起こし保存と患者同意

文字起こしの永続保存を開始した時点で、会話テキストは診療情報、すなわち要配慮個人情報の保存にあたる。本実装の患者向け初回同意フローへ、会話内容をテキストで記録する旨を追加する。

設計する導線は次のとおりとし、法務確定前の UI 文言はプレースホルダにする。

1. 患者が初めて診療リンクを開いた際、入室前に記録について説明する。
2. 「会話内容をテキスト記録する」旨の独立した同意チェックを表示する。
3. チェックされるまでゲスト参加を進めない。
4. 同意しない場合は参加させず、代替手段の案内へ遷移する。

プレースホルダ例: `[会話内容のテキスト記録に関する説明・利用目的・保存期間をここに表示]`。これは最終的な同意文言や法的評価ではない。

## Soniox の待機時課金

Soniox セッションは pause 中も課金される。患者待ち、長時間の中断、待機画面では pause のまま保持せず、Soniox WebSocket を切断する。診療再開時に新しい一時 API key を取得して再接続し、必要な one_way target と `stt-rt-v5` を再設定する。

## このリポとの境界

このリポは実在患者の利用を想定せず、音声・字幕・会話ログを永続保存しない。同意管理も実装しない。本書は本実装リポが永続化を導入する前に満たすべき注意事項である。
