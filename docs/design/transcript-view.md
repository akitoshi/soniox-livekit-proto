# 診療後トランスクリプト表示・コピー設計

## 画面と取得単位

診療終了後、または医師用ダッシュボードから `session_id` 単位のページを開き、確定済み会話全文を時系列で表示する。取得前に Clerk 認証を確認し、Supabase RLS とサーバー側認可の双方で、その `session_id` の `doctor_user_id` が閲覧中の医師本人であることを検証する。URL の `session_id` は外部入力として検証し、存在の有無を越権ユーザーへ漏らさない。

表示は次のプレーンテキスト表現を基準とする。

```text
[医師 ja] こんにちは、今日はどうされましたか
[患者 de→ja] 昨日から喉が痛いです (原文: Ich habe seit gestern Halsschmerzen)
```

患者発話は翻訳文を本文にし、原文を括弧内に併記する。翻訳がない場合も原文だけで破綻しない形式にする。DB 取得順は `spoken_at` の昇順を基本とし、同値時の安定した tie-breaker を本実装で定める。

## 全文コピー

「全文コピー」は、画面に表示した全確定セグメントを話者・言語ラベル付きの装飾なしプレーンテキストへ整形し、クリップボードへ入れる。HTML、Markdown、患者識別情報など画面外の情報は混ぜない。コピー結果はカルテシステムへそのまま貼り付けられる改行区切りとする。

## 任意の SOAP 下書き

SOAP 下書き生成は任意機能とする。採用する場合、ブラウザから LLM へ直接送信せず、Next.js API Route が Clerk 認証と担当診療の認可を再確認してから全文を1回送る。LLM の資格情報はサーバーだけに置く。出力は「下書き」であり、医師が確認・修正したうえで専用ボタンからプレーンテキストとしてコピーできるようにする。トランスクリプト画面と同じアクセス制御を適用する。
