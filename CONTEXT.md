## 用語

### 診療ルーム(consultation room)
LiveKit の room。ルーム名を知っている者は誰でも参加できる検証用空間。認証・本人確認はない(プロトタイプの意図的制約)。

### ロール(role)
参加者の属性。`doctor`(医師)または `patient`(患者)。ロビーでの自己申告で決まり、認証で裏付けない。想定は医師=日本語話者、患者=外国語話者の1対1。

### 患者言語(patient language)
患者が字幕として受け取りたい言語。Soniox 対応言語(60言語)から選択する。医師セッションの one_way 翻訳ターゲットとして使う。医師がロビーで手動選択する(2026-07-13 決定。本実装では患者選択の自動連携 — docs/design/production-entry-flow.md 参照)。

### one_way ×2(話者ごとの一方向翻訳)
各参加者が自分の Soniox セッションに `translation: { type: "one_way", target_language: <相手の言語> }` を設定する方式。医師セッション target=患者言語、患者セッション target=`ja`。one_way は話された**全言語**(文中の英語混在を含む)をターゲット言語へ翻訳する(公式 docs 検証済み 2026-07-13)。two_way は設定2言語以外(特に英語)が翻訳されないため使用しない。

### 原文トークン / 翻訳トークン
Soniox トークンの `translation_status` による区別。`"original"` / `"none"` = 原文(音声位置の start_ms/end_ms あり)、`"translation"` = 翻訳(**タイムスタンプなし**)。UI は原文と翻訳の1対1対応を仮定してはならない。

### 発話(utterance)
`<end>` エンドポイントトークン検出(または 2.5 秒フォールバックタイマー)で確定する字幕の単位。確定字幕(final)は reliable、非確定(interim)は lossy で配信される。

### 字幕ペイロード(CaptionPayload)
LiveKit data channel(topic: `captions`)で参加者間配信される JSON。受信側は必ずスキーマ検証(`isCaptionPayload`)を通す。外部入力として扱う。

### 会話ログ(session transcript)
確定字幕のメモリ内バッファ。コピー機能専用。**このリポでは永続化しない**(reload で消えるのが正しい挙動)。永続化の設計は docs/design/transcript-persistence.md(本実装向け・実装しない)。

### Soniox 対応言語定数
`lib/languages.ts`(owt-2 で新設)。公式 Supported languages 準拠の60言語。**zh は単一エントリで、簡体/繁体・広東語の区別は Soniox に存在しない**(公式表で検証済み 2026-07-13)。プログラム上の正は Get models API。言語コードの推測記述は禁止。

## 変換ルール

言語コード → 表示名の対応は `lib/languages.ts` を単一情報源とし、全行をテーブルテストで固定する。

| 内部 | 外部(表示) |
|---|---|
| ISO コード(例: `de`) | 自国語表記(例: `Deutsch`)+日本語表記(例: ドイツ語) |

- 未知の言語コードのフェイルセーフ: **コードをそのまま表示し、選択肢には出さない**(安全側)

## 公開してよい / してはいけない情報の線引き

- **公開してよい**(参加者間・画面上): 表示名、ロール、選択言語、字幕テキスト(原文・翻訳)、参加状態
- **公開してはいけない**:
  - `SONIOX_API_KEY` / `LIVEKIT_API_SECRET`(サーバー専用。クライアントバンドル・ログ・エラーメッセージに出さない)
  - Soniox 一時キー(ログに出さない。60秒失効・単一使用を維持)
  - 字幕・会話ログの外部送信・永続保存(このリポでは localStorage 含め一切禁止)
  - 実在する患者の情報(検証用リポ。README の注意書きを維持)
